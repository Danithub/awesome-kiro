# Kiro 명령어 자동 실행 허용 정책 안내

Kiro가 프롬프트 중에 터미널 명령을 실행할 때, **어떤 명령을 자동으로 허용할지**를
정하는 정책이 어디에 저장되는지 정리한 안내 문서입니다.

이 문서는 Kiro를 쓰는 사람이 읽기 위한 설명서입니다. 편하게 훑어보시고, 필요한
부분만 찾아 참고하시면 됩니다.

> 참고: 이 내용은 Kiro IDE 설정(Settings) 화면에서는 찾을 수 없어, 별도로 정리해
> 둔 문서입니다.

## 1. 명령어 자동 실행은 어떻게 정해지나요

Kiro가 터미널 명령을 실행하려고 할 때, 미리 정해둔 **허용 규칙**과 명령을
비교합니다. 결과는 세 가지 중 하나입니다.

- **허용(allow) 규칙에 맞으면** → 물어보지 않고 바로 실행합니다.
- **거부(deny) 규칙에 맞으면** → 실행을 막습니다.
- **어느 규칙에도 안 맞으면** → 실행해도 되는지 여러분에게 한 번 물어봅니다.

규칙은 `python *`처럼 **명령 패턴**으로 적어둡니다. 여기서 `*`는 "뒤에 무엇이 오든
상관없음"을 뜻합니다. 예를 들어 `python *` 규칙이 있으면 `python app.py`,
`python -m pip install ...` 같은 명령이 모두 자동으로 허용됩니다.

한 가지만 기억하시면 됩니다. 같은 명령에 허용과 거부가 동시에 걸리면, 안전을 위해
**거부가 항상 이깁니다.**

## 2. 정책은 어디에 저장되나요

명령을 허용할 때 Kiro는 "**어디까지 적용할까요?**"를 묻습니다. 여기서 고른 범위에
따라 규칙이 저장되는 파일 위치가 달라집니다.

- **전체 워크스페이스에 적용** → 내 컴퓨터의 모든 프로젝트에서 공통으로 쓰이는
  규칙입니다.
- **이 워크스페이스에만 적용** → 지금 열려 있는 이 프로젝트에서만 쓰이는 규칙입니다.

아래 표에서 `{사용자}`는 Windows 계정 이름, `{프로젝트 hash}`는 프로젝트 폴더마다
자동으로 붙는 고유 식별자입니다.

| 적용 범위 | 저장되는 파일 위치 |
|-----------|--------------------|
| 전체 워크스페이스 | `C:\Users\{사용자}\.kiro\settings\permissions.yaml` |
| 이 워크스페이스 | `C:\Users\{사용자}\.kiro\workspace-roots\{프로젝트 hash}\permissions.yaml` |

두 곳에 같은 명령 규칙이 있으면, 더 좁은 범위인 **이 워크스페이스 규칙이 우선**
적용됩니다.

## 3. 정책 파일 직접 열어보기

앞에서 본 `permissions.yaml` 파일을 직접 열면 규칙을 눈으로 확인하거나 손으로
고칠 수 있습니다. 파일은 아래처럼 생겼습니다.

```yaml
rules:
  - capability: shell
    effect: allow
    match:
      - python *
      - Remove-Module *
      - powershell *
```

각 항목이 뜻하는 바는 이렇습니다.

- `capability` : 규칙이 적용되는 대상. 터미널 명령이면 `shell`입니다.
- `effect` : `allow`(자동 허용) 또는 `deny`(차단).
- `match` : 어떤 명령에 적용할지 적는 패턴 목록. `*`는 임의의 뒷부분을 뜻합니다.

예를 들어 `Remove-Module *` 규칙이 있으면
`Remove-Module PSReadLine -ErrorAction SilentlyContinue` 같은 명령이 여기에
맞아떨어져서, 물어보지 않고 바로 실행됩니다.

## 4. 지금 이 프로젝트에 걸린 규칙

참고로, 현재 이 프로젝트(이 워크스페이스)에는 아래 세 가지 셸 허용 규칙이
등록되어 있습니다.

| 규칙 패턴 | 자동 허용되는 명령 |
|-----------|--------------------|
| `python *` | 파이썬 실행 관련 명령 |
| `Remove-Module *` | PSReadLine 정리 등 모듈 제거 명령 |
| `powershell *` | powershell 하위 셸 호출 |

새로운 명령을 처음 실행할 때 Kiro가 허용 여부를 물어보면, "이 워크스페이스에만
허용"을 고르는 순간 위 목록에 규칙이 한 줄 더 추가되는 방식입니다.
