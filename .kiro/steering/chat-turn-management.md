# 채팅 턴 관리 및 세션 전환 안내 규칙

## 매 응답 말미에 표시할 것
모든 응답의 가장 마지막에 아래 형식으로 "턴 상태" 한 줄을 붙인다.

> 🔄 **턴 상태:** 현재 약 N번째 턴 / 권장 한도 약 {limit}턴 — {상태메시지}

- `N`: 이 대화에서 보이는 사용자 메시지 수를 기준으로 한 **대략적인 추정값**.
  정확한 카운트가 아님을 전제로 하며, 확신이 없으면 "약" 표기를 유지한다.
- 컨텍스트가 요약/압축되어 초반 대화가 보이지 않을 수 있으므로,
  숫자가 어긋날 수 있다는 점을 인지하고 과신하지 않는다.

## 권장 한도(limit) 기준
대화의 성격에 따라 권장 한도를 다르게 적용한다.

- **요구사항/설계 등 토론 위주 단계:** 약 8~12턴
- **Spec 구현(tasks) 단계:** 턴 수보다 "작업 단위" 우선 →
  tasks 항목 1~3개(또는 하나의 응집된 기능 묶음)를 끝낼 때마다 전환 권장
- **단순 Q&A / 디버깅:** 약 10~15턴

## 상태메시지 규칙
- 한도의 70% 미만: "여유 있음 — 계속 진행하셔도 좋습니다."
- 한도의 70~99%: "슬슬 정리 시점이 가까워집니다."
- 한도 도달/초과: "지금까지 내용을 정리하고 **새 채팅으로 넘어가시길 권장**합니다."

## 새 채팅 권장 시 함께 안내할 것
한도에 도달했거나, spec의 한 단계(requirements/design/tasks)가 끝났을 때는
다음을 함께 제안한다.

1. **"지금까지 합의/완료된 내용을 spec 파일(.md)에 반영하고 마무리할까요?"** 라고 확정 제안
2. 확정 후, 새 채팅에서 해당 spec 파일을 참조해 이어가면 맥락 손실 없이 진행됨을 안내
3. 단계 경계(요구사항→설계→구현)에서는 턴 수와 무관하게 새 채팅 전환을 우선 권장

## 예외
- 사용자가 명시적으로 "계속하자"고 하면 한도를 넘겨도 중단을 강요하지 않고,
  대신 안내 문구만 가볍게 유지한다.
