# Guide 키워드 라우팅 규칙

## 목적
- `.kiro/guides/` 폴더의 통합 가이드들은 평소에는 로드하지 않는(`inclusion: manual`) 참조
  문서다. 특정 유형의 작업에서만 필요하다.
- 이 규칙은 사용자 요청에 아래 **트리거 키워드**가 나타나면, 해당 guide를 **먼저 읽고
  그 방법론을 따르도록** 강제한다.

## 동작 방식
1. 사용자 요청/맥락에서 아래 매핑 표의 키워드와 일치하는 신호를 감지한다.
2. 일치하는 guide 파일을 `.kiro/guides/<파일명>`에서 **읽는다**(read). 사용자가 직접
   `#guide-name`으로 부르지 않아도, 관련 작업이면 스스로 읽어 참고한다.
3. 해당 guide의 핵심 원칙·정확성 속성·체크리스트를 준수하여 작업을 수행한다.
4. 어떤 guide를 참고했는지 응답에서 한 줄로 밝힌다.
   (예: "샘플 가이드 작업이라 `sample-guide`를 참고했습니다.")
5. 여러 guide가 동시에 해당하면 모두 읽고, 겹치는 원칙은 더 구체적인 guide를 우선한다.
   어떤 guide도 확실히 맞지 않으면 임의로 읽지 말고 평소대로 진행한다.

## 키워드 → Guide 매핑

아래는 형식을 보여주는 **예시**다. 실제 프로젝트에서는 이 표에 자신의 guide와 트리거
키워드를 한 줄씩 추가해 쓴다. (guide 작성 형식은 `.kiro/guides/sample-guide.md` 참고)

| Guide 파일 | 트리거 키워드 |
|-----------|--------------|
| `sample-guide.md` | (예시) 샘플, 템플릿, 가이드 작성, guide 형식 |

## 주의
- 키워드는 판단을 돕는 신호일 뿐이며, 문자 그대로의 완전 일치를 요구하지 않는다.
  의미가 통하면 유의어·영문/한글 표기 차이도 매칭으로 본다.
- guide를 읽는 것은 맥락 참고를 위한 것이며, guide에 없는 프로젝트별 지시가 있으면
  사용자의 최신 지시를 우선한다.
