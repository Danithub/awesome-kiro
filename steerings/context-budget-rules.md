# 컨텍스트 폭증 방지 규칙 (대용량 파일 읽기 / 광역 grep)

한 턴의 입출력이 컨텍스트 한도를 넘겨 응답이 중단되는 문제를 예방하기 위한
**강제 규칙**입니다. 편집(str_replace 등) 자체는 성공하더라도, **읽기/검색 단계
에서 컨텍스트가 부풀면 턴이 끊깁니다.** 아래 규칙은 선택이 아니라 의무입니다.

## 목차
1. 핵심 원칙 (왜 끊기는가)
2. 대용량 파일 읽기 규칙 (전체 재읽기 금지)
3. grep / 검색 출력 폭발 방지 규칙
4. 반복 작업 시 재읽기 최소화
5. 금지 사항 (Anti-patterns)
6. 실행 체크리스트

## 1. 핵심 원칙 (왜 끊기는가)

- 턴 중단의 주원인은 코드 실패가 아니라 **한 턴에 쌓인 입출력 토큰이 한도를
  초과**하는 것이다.
- 두 가지 습관이 겹치면 폭증한다:
  1. **대용량 파일(수백~수천 줄)을 매 턴 통째로 재읽기** → 매번 수만 토큰 유입
  2. **광역 grep의 에코 아티팩트** → PSReadLine 렌더링 특성상 같은 줄이 수십 번
     중복 출력되어 출력이 폭발
- 따라서 "일단 다 읽고 판단하자", "전체를 훑어보자"는 접근을 **버린다.** 항상
  **필요한 최소 범위만** 읽고 검색한다.

## 2. 대용량 파일 읽기 규칙 (전체 재읽기 금지)

- **약 300줄을 넘는 파일은 전체를 한 번에 읽지 않는다.** 먼저 구조만 파악하고
  필요한 구간만 범위 지정으로 읽는다.
- 절차:
  1. `read_code`(선택자/시그니처) 또는 `grep_search`로 **위치(줄 번호)만** 먼저 찾음
  2. 찾은 지점 주변만 `read_file`의 `start_line`/`end_line`으로 **범위 읽기**
  3. 편집 대상 구간이 명확하면 그 구간만 읽고 바로 `str_replace`
- **같은 파일을 매 턴 다시 통째로 읽지 않는다.** 한 번 읽어 파악한 구조는 재사용
  하고, 바뀐 부분만 좁게 다시 읽는다.
- XML/로그/생성물 등 **초대용량 파일(예: 2,000줄 이상)** 은 전체 읽기를 절대
  금지하고, 반드시 grep으로 앵커를 잡은 뒤 ±수십 줄만 읽는다.

## 3. grep / 검색 출력 폭발 방지 규칙

- **파일 내용 검색은 셸 grep이 아니라 `grep_search` 도구를 사용한다.** 이 도구는
  구조화된 결과(파일/줄번호/제한된 컨텍스트)를 주며 에코 중복이 없다.
- 셸에서 `Select-String`/`findstr`로 광역 검색을 돌리지 않는다. PSReadLine
  렌더링 때문에 같은 줄이 수십 번 중복 출력되어 컨텍스트가 폭발한다.
- 광역 패턴(예: `[가-힣]` 전수 매칭)은 **금지**한다. 반드시 아래로 좁힌다:
  - `includePattern`으로 **파일 유형/경로 한정** (예: `**/*.md`)
  - 패턴을 **구체적인 문자열/식별자**로 좁힘 (전수 매칭 금지)
  - 결과는 도구가 최대 100건으로 제한하지만, 그 전에 스스로 범위를 줄인다
- 검색 결과가 여전히 많으면 **패턴을 더 구체화**해 재검색한다. "일단 넓게 잡고
  눈으로 거른다"는 접근을 금지한다.

## 4. 반복 작업 시 재읽기 최소화

- 같은 파일을 여러 번 수정할 때, **매 수정마다 전체를 다시 읽지 않는다.**
- `str_replace`는 파일 전체를 읽지 않고도 국소 편집이 가능하다. 편집할 앵커
  (앞뒤 2~3줄)만 알면 되므로, 필요한 최소 컨텍스트만 확보한다.
- 편집 검증도 전체 재읽기 대신 **바뀐 구간 주변만** 범위 읽기로 확인한다.

## 5. 금지 사항 (Anti-patterns)

- 🚫 300줄 초과 파일을 범위 지정 없이 **전체 읽기**
- 🚫 같은 대용량 파일을 **매 턴 통째로 재읽기**
- 🚫 셸 `Select-String`/`findstr`/`grep`으로 **광역 검색** (에코 폭발)
- 🚫 `[가-힣]` 같은 **전수 매칭** 패턴으로 검색
- 🚫 `includePattern` 없이 워크스페이스 전체를 대상으로 무제한 검색
- 🚫 "넓게 잡아 눈으로 거르기" — 좁혀서 검색하지 않는 태도

## 6. 실행 체크리스트

- [ ] 파일이 300줄을 넘으면 grep/시그니처로 위치부터 찾았는가
- [ ] 그 위치 주변만 범위(start_line/end_line)로 읽었는가
- [ ] 파일 검색에 셸 grep이 아니라 `grep_search` 도구를 썼는가
- [ ] 검색 패턴을 구체 문자열로 좁히고 `includePattern`으로 한정했는가
- [ ] 전수 매칭(`[가-힣]` 등) 광역 패턴을 피했는가
- [ ] 같은 파일을 불필요하게 전체 재읽기하지 않았는가
