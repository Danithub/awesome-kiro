# 서브에이전트 위임 실패 시 폴백 규칙

## 원칙
- 서브에이전트(invoke_sub_agent) 위임은 "가능하면 활용"하는 최적화 수단일 뿐,
  작업 진행의 필수 조건이 아니다.
- 위임 호출이 실패하거나 즉시 취소되면, 지체 없이 해당 작업을 **직접 수행**한다.
  사용자에게 다시 위임을 시도할지 되묻느라 진행을 멈추지 않는다.

## 판단 기준
- 동일 세션에서 서브에이전트 호출이 **1회라도 즉시 취소/실패**하면,
  그 세션에서는 위임이 고장난 것으로 간주하고 이후 작업은 직접 처리한다.
- 페이로드(컨텍스트·프롬프트) 크기를 줄여도 취소가 반복되면,
  이는 입력 문제가 아니라 세션/환경의 실행 메커니즘 문제로 판단한다.

## 직접 수행 방식
- context-gatherer가 하던 일(코드 탐색·파일 분석·관련 위치 파악)은
  grep/find/read 등 파일 도구로 직접 조사한다.
- general-task-execution이 하던 일(구현·수정·검증)은
  현재 세션에서 직접 코드를 작성하고 검증한다.

## 사용자 안내
- 위임이 실패해 직접 수행으로 전환할 때는, 한 줄로 그 사실만 간단히 알린다.
  (예: "위임이 취소되어 직접 처리하겠습니다.")
- 반복적으로 실패하면, 근본 해결을 위해 새 세션/IDE 재시작을 함께 권고한다.
